# Preferences and profiles

## English

Morph UI uses a structured preference model instead of a single freeform prompt.

### Profile shape

A profile includes:

- profile name
- natural language instruction
- density
- typography scale
- content width
- contrast preference
- theme preference
- distraction hiding
- primary content emphasis
- repetitive chrome collapse
- list versus card preference
- sticky summary and sticky TOC
- navigation simplification
- click target size preference
- motion reduction
- secondary content placement
- brand preservation strictness
- accessibility priority
- aggressiveness

### Current profile scopes

- global default profile
- site-level override
- data model support for path-level override
- learned adjustments based on feedback

### Seeded profiles

- Reader Focus
- Dense Catalog
- Calm Dashboard
- Docs Navigator
- Accessible Contrast

## 한국어

Morph UI는 단일 자유 입력 프롬프트 대신 구조화된 preference 모델을 사용합니다.

### 프로필 구조

프로필에는 다음이 포함됩니다.

- 프로필 이름
- 자연어 지시문
- density
- typography scale
- content width
- contrast preference
- theme preference
- distraction 숨기기
- 핵심 콘텐츠 강조
- 반복 chrome 축소
- list/card 선호
- sticky summary, sticky TOC
- navigation 단순화
- 클릭 타깃 크기 선호
- 애니메이션 감소
- 보조 콘텐츠 위치 선호
- 브랜드 보존 강도
- 접근성 우선순위
- aggressiveness

### 현재 지원 범위

- 전역 기본 프로필
- 사이트 단위 override
- path 단위 override를 위한 데이터 모델 지원
- feedback 기반 learned adjustment

### 시드 프로필

- Reader Focus
- Dense Catalog
- Calm Dashboard
- Docs Navigator
- Accessible Contrast
