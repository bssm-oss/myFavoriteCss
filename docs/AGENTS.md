# Documentation agent instructions

## English

These instructions apply to `docs/`.

### Core responsibility

Documentation in this repository is part of the product operating surface, not an afterthought.

### Hard rules

- keep docs aligned with implemented behavior
- keep docs bilingual with `## English` and `## 한국어`
- prefer concrete statements over marketing language
- link related docs when a change creates a new operational dependency

### File maintenance expectations

- update `docs/README.md` when adding or removing major docs
- keep release, environment, diagnostics, and workflow docs cross-linked
- do not claim unsupported provider capabilities
- keep privacy wording accurate and specific

### Style

- use short sections
- keep lists flat
- include commands and file paths when useful
- explain what changed, why it matters, and where to look next

## 한국어

이 지침은 `docs/`에 적용됩니다.

### 핵심 책임

이 저장소의 문서는 부가물이 아니라 제품 운영 surface의 일부입니다.

### 강한 규칙

- 문서는 실제 구현 동작과 맞아야 합니다.
- 문서는 `## English`, `## 한국어` 섹션을 유지합니다.
- 마케팅 문구보다 구체적인 설명을 사용합니다.
- 새 운영 의존성이 생기면 관련 문서를 서로 링크합니다.

### 파일 유지보수 기대사항

- 주요 문서를 추가/삭제하면 `docs/README.md`를 갱신합니다.
- release, environment, diagnostics, workflow 문서는 서로 연결 상태를 유지합니다.
- 미지원 provider capability를 지원하는 것처럼 쓰지 않습니다.
- privacy 문구는 정확하고 구체적으로 유지합니다.

### 스타일

- 섹션은 짧게 유지합니다.
- 리스트는 평평하게 유지합니다.
- 필요하면 명령어와 파일 경로를 넣습니다.
- 무엇이 바뀌었는지, 왜 중요한지, 다음에 어디를 봐야 하는지 설명합니다.
