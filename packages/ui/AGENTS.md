# UI package agent instructions

## English

These instructions apply to `packages/ui`.

- keep primitives reusable across extension and web app
- keep accessible defaults
- do not encode product-specific business logic in shared UI primitives
- when visible labels or structural primitives change, review side panel and docs screenshots or notes

## 한국어

이 지침은 `packages/ui`에 적용됩니다.

- 프리미티브는 extension과 웹앱 모두에서 재사용 가능해야 합니다
- 접근성 기본값을 유지합니다
- shared UI primitive 안에 제품별 비즈니스 로직을 넣지 않습니다
- 보이는 라벨이나 구조 프리미티브가 바뀌면 side panel과 관련 문서 노트도 함께 검토합니다
