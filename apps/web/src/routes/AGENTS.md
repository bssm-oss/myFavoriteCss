# Web routes agent instructions

## English

These instructions apply to `apps/web/src/routes`.

- route files should stay presentational and deterministic
- help/privacy routes should describe real extension behavior
- fixture routes should keep stable headings and major blocks unless there is a reason to change them

## 한국어

이 지침은 `apps/web/src/routes`에 적용됩니다.

- route 파일은 표현 중심이고 deterministic해야 합니다
- help/privacy route는 실제 extension 동작을 설명해야 합니다
- fixture route는 바꿀 이유가 없다면 stable heading과 주요 block을 유지합니다
